# AWX

## Overview

AWX is deployed by the AWX Operator and exposed through Traefik. The cloud lane
default is host-root access at `https://awx.dmf.example.com/` on the private
Traefik lane. Path mode at `/awx` remains as a fallback for non-DNS lanes.

Human access uses Authentik SAML. AWX Generic OIDC is intentionally not used for
admin SSO because AWX 24.6.1 documents Generic OIDC as authentication-only and
without team/organization authorization mapping.

Automation access remains local AWX admin/service-token based. Browser SAML is
not used for Ansible API bootstrap or day-2 integration tasks.

## Key configuration

File: `k3s-lab-bootstrap/roles/stack/operator/awx/templates/awx-instance.yml.j2`

Host mode `extra_settings` include:

- `TOWER_URL_BASE = 'https://awx.dmf.example.com'`
- `STATIC_URL = '/static/'`
- `CSRF_COOKIE_PATH = '/'`
- `SESSION_COOKIE_PATH = '/'`

SAML `extra_settings` include:

- `SOCIAL_AUTH_SAML_SP_ENTITY_ID = 'awx'`
- `SOCIAL_AUTH_SAML_ENABLED_IDPS.authentik.url =
  'https://auth.dmf.example.com/application/saml/awx/sso/binding/redirect/'`
- `SOCIAL_AUTH_SAML_ENABLED_IDPS.authentik.entity_id =
  'https://awx.dmf.example.com/sso/metadata/saml/'`
- `SOCIAL_AUTH_SAML_ENABLED_IDPS.authentik.x509cert` from Authentik's signing
  certificate, discovered at AWX deploy time
- `SOCIAL_AUTH_SAML_USER_FLAGS_BY_ATTR` maps SAML group `ops-admin` to AWX
  `is_superuser`
- `LOGIN_REDIRECT_OVERRIDE = '/sso/login/saml/?idp=authentik'`

Important: scalar Python string values in `extra_settings` must include nested
quotes, for example `value: "'awx'"`. Dict values should be emitted as YAML/JSON
objects so the AWX Operator writes valid Python settings.

## Authentik wiring

File:
`k3s-lab-bootstrap/roles/stack/operator/authentik/templates/blueprints/20-app-providers.yaml.j2`

The Authentik blueprint creates the AWX application with a SAML provider:

- Provider name: `AWX SAML`
- Application slug: `awx`
- ACS URL: `https://awx.dmf.example.com/sso/complete/saml/`
- Audience: `awx`
- Issuer: `https://awx.dmf.example.com/sso/metadata/saml/`
- Binding: `post`
- Signing key: `authentik Self-signed Certificate`
- Property mappings: username, uid, email, name, groups

The AWX role reads the Authentik signing certificate after Authentik is deployed
and injects the stripped certificate body into AWX SAML settings. Do not
hardcode generated Authentik certificate material in repo defaults.

## API automation

The AWX integration role uses local/API credentials:

- Bootstrap admin user: `awx-local-admin`
- Bootstrap admin password source: `vault_awx_admin_password`
- Service account/token: generated by `awx-integration` and persisted through
  the configured vault/OpenBao path

This is deliberate. SAML is for human browser login only.
The local bootstrap admin username is intentionally different from the
Authentik human operator username (configured via
`authentik_bootstrap_passkey_username`); AWX SAML/enterprise users can
conflict with pre-existing local users of the same username.

From the env repo:

```bash

./bin/run-playbook.sh hetzner-arm ../dmf-infra/k3s-lab-bootstrap/playbooks/693-awx-integration.yml
```

## Verification

```bash
# Pods and CR
sudo k3s kubectl get awx,pods -n awx

# SAML settings inside AWX
sudo k3s kubectl -n awx exec deploy/awx-web -- \
  awx-manage print_settings SOCIAL_AUTH_SAML_ENABLED_IDPS

# SSO redirect
curl -Ik https://awx.dmf.example.com/sso/login/saml/?idp=authentik

# API auth must still work with local credentials
curl -u awx-local-admin:<admin-password> https://awx.dmf.example.com/api/v2/me/
```

## Common failure modes

- AWX API returns `401` during integration:
  - Check the integration role is authenticating as `awx-local-admin`, not `admin`.
  - Check the database password was synced with `awx-manage update_password`.
- AWX SAML redirects but login is not admin:
  - Confirm the Authentik user belongs to `ops-admin`.
  - Confirm the SAML groups attribute is present in the assertion.
  - Confirm AWX `SOCIAL_AUTH_SAML_USER_FLAGS_BY_ATTR` maps that group.
- AWX web crashes with `SyntaxError` in `/etc/tower/settings.py`:
  - Check `extra_settings` quote handling. Python string values need nested
    quotes; dict values must render as valid Python objects.
- Path-mode static assets fail:
  - Check `awx-redirect`, `awx-strip-prefix`, and the path-mode cookie/static
    settings in the AWX CR template.
